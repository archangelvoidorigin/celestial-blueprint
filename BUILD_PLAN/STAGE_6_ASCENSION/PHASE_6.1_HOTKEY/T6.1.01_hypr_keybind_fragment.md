# TASK T6.1.01 — `hypr/celestial.conf` keybind fragment + `hypr/link_hypr.sh`

- **Stage / Phase**: Stage 6 — Ascension / Phase 6.1
- **Depends on**: T1.8.01, T3.1.04 (`cb` CLI must be installed)
- **Produces**: `hypr/celestial.conf`, `hypr/link_hypr.sh`

## Context

Celestial Blueprint's OS integration lives inside the project directory — configs are never scattered to `~/.config` directly. For Hyprland, the pattern is a project-local `.conf` fragment that gets sourced from the user's `hyprland.conf`. This task creates the fragment (keybinds only; window rules come in T6.2.01) and a script that appends one `source` line to `~/.config/hypr/hyprland.conf` if not already present.

Two keybinds:
- `$mainMod + P` → floating terminal showing `cb blueprint list`, then dropped into an interactive shell (requires `cb` on PATH from Stage 3)
- `$mainMod + SHIFT + P` → open frontend at `http://localhost:5173` in default browser

The floating-terminal behaviour (size, center, class rule) is set by the windowrule added in T6.2.01. The window class used is `celestial-plan` — the `--class` flag passed to the terminal emulator.

## Objective

Create `hypr/celestial.conf` with the two keybind directives and `hypr/link_hypr.sh` that sources it from `hyprland.conf`.

## Instructions

1. Create the `hypr/` directory at the project root:
   ```bash
   mkdir -p /home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/hypr
   ```

2. Create `hypr/celestial.conf` with exactly:
   ```ini
   # Celestial Blueprint — Hyprland keybinds
   # Source this file from ~/.config/hypr/hyprland.conf:
   #   source = /home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/hypr/celestial.conf
   #
   # Requires: cb CLI on PATH (Stage 3), alacritty installed.
   # To use a different terminal: replace `alacritty` with your emulator
   # and ensure --class and -e (or equivalent) flags are supported.

   # $mainMod + P — floating terminal: show blueprint list, then interactive shell.
   # `cb blueprint create "<name>"` creates one; `exec fish` keeps the shell open.
   bind = $mainMod, P, exec, alacritty --class celestial-plan --title "Celestial Plan" -e fish -c "cb blueprint list; echo; echo 'Celestial Blueprint — try: cb blueprint create \"My Plan\"'; exec fish"

   # $mainMod + SHIFT + P — open frontend in default browser
   bind = $mainMod SHIFT, P, exec, xdg-open http://localhost:5173
   ```

3. Create `hypr/link_hypr.sh` with exactly:
   ```bash
   #!/usr/bin/env bash
   # Appends a source line to ~/.config/hypr/hyprland.conf if not already present.
   # Run once after cloning / installing Celestial Blueprint.
   # Override the target dir for testing with CELESTIAL_HYPR_DIR.
   set -euo pipefail

   CONF_DIR="${CELESTIAL_HYPR_DIR:-$HOME/.config/hypr}"
   CONF_FILE="$CONF_DIR/hyprland.conf"
   FRAGMENT="/home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/hypr/celestial.conf"
   SOURCE_LINE="source = $FRAGMENT"

   mkdir -p "$CONF_DIR"
   touch "$CONF_FILE"

   if grep -qF "$SOURCE_LINE" "$CONF_FILE"; then
       echo "Already sourced in $CONF_FILE — nothing to do."
   else
       echo "" >> "$CONF_FILE"
       echo "# --- Celestial Blueprint ---" >> "$CONF_FILE"
       echo "$SOURCE_LINE" >> "$CONF_FILE"
       echo "Added source line to $CONF_FILE"
       echo "Reload Hyprland (hyprctl reload) to activate."
   fi
   ```

4. Make the script executable:
   ```bash
   chmod +x /home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/hypr/link_hypr.sh
   ```
