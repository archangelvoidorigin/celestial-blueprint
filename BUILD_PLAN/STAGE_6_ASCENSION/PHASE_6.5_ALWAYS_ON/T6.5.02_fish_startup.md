# TASK T6.5.02 — Fish startup notification snippet

- **Stage / Phase**: Stage 6 — Ascension / Phase 6.5
- **Depends on**: T6.5.01
- **Produces**: `tools/celestial_startup.fish`, edit to `docs/automation.md`

## Context

Fish runs `~/.config/fish/config.fish` at every new session. A startup check that silently passes when the backend is healthy and fires a desktop notification when it's down is useful — it catches situations where the systemd service failed overnight. The snippet lives in the project as `tools/celestial_startup.fish` (not edited directly into `~/.config/fish/config.fish` — that file belongs to `my-arch-setup`). Instructions tell the user to `source` it from their config.fish.

## Objective

Create `tools/celestial_startup.fish` and document the setup in `docs/automation.md`.

## Instructions

1. Create `tools/celestial_startup.fish` with exactly:
   ```fish
   # Celestial Blueprint — Fish startup check
   # Source this from ~/.config/fish/config.fish:
   #   source /home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/tools/celestial_startup.fish
   #
   # Fires a dunst/notify-send notification if the backend service is down.
   # Runs silently if all is well.

   function _celestial_startup_check --on-event fish_greeting
       if not systemctl --user is-active --quiet celestial-backend 2>/dev/null
           set msg "Celestial Blueprint backend is DOWN. Run: systemctl --user start celestial-backend"
           if command -q dunstify
               dunstify -u critical -t 12000 "Celestial Blueprint Offline" $msg
           else if command -q notify-send
               notify-send -u critical "Celestial Blueprint Offline" $msg
           end
           echo "⚠ $msg" >&2
       end
   end
   ```

2. Edit `docs/automation.md`: append a `## Stage 6 — Startup Check` section:
   ```markdown
   ## Stage 6 — Startup Check

   Source the Fish startup check so every new terminal session verifies the backend is running:

   ```fish
   # Add to ~/.config/fish/config.fish (once):
   source /home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/tools/celestial_startup.fish
   ```

   If the backend is down, a dunst notification fires and an error is printed to stderr. Start the service with:

   ```bash
   systemctl --user start celestial-backend
   journalctl --user -u celestial-backend -n 30   # see why it failed
   ```
   ```
