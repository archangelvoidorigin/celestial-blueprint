# TASK T6.5.01 — `systemd/celestial-backend.service` + `link_service.sh`

- **Stage / Phase**: Stage 6 — Ascension / Phase 6.5
- **Depends on**: T1.8.01
- **Produces**: `systemd/celestial-backend.service`, `link_service.sh`

## Context

Systemd user services live at `~/.config/systemd/user/`. Per the project rule (everything inside CHAOS_DAO_FORGE), the unit file lives at `Celestial_Blueprint/systemd/celestial-backend.service` and is symlinked by `link_service.sh`. The service uses the backend's own venv (`backend/.venv/bin/uvicorn`) and reads `backend/.env` via `EnvironmentFile`. It targets `graphical-session.target` (starts after the desktop session is up), restarts on failure, and keeps logs in the journal (`journalctl --user -u celestial-backend`).

## Objective

Create the systemd user service unit and a script that installs it (symlink + enable + start).

## Instructions

1. Create the `systemd/` directory at the project root:
   ```bash
   mkdir -p /home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/systemd
   ```

2. Create `systemd/celestial-backend.service` with exactly:
   ```ini
   [Unit]
   Description=Celestial Blueprint Backend
   Documentation=file:///home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/docs/API.md
   After=graphical-session.target network.target
   Wants=graphical-session.target

   [Service]
   Type=simple
   WorkingDirectory=/home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/backend
   ExecStart=/home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/backend/.venv/bin/uvicorn app.main:app --port 8787 --host 127.0.0.1
   EnvironmentFile=-%h/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/backend/.env
   Restart=on-failure
   RestartSec=5
   StandardOutput=journal
   StandardError=journal
   SyslogIdentifier=celestial-backend

   [Install]
   WantedBy=default.target
   ```

   Note: `EnvironmentFile=-%h/...` uses `-%h` to set the file relative to `$HOME` and the leading `-` makes it optional (service starts even if `.env` is absent). `--host 127.0.0.1` binds locally only — use `0.0.0.0` if remote access is needed.

3. Create `link_service.sh` at the project root with exactly:
   ```bash
   #!/usr/bin/env bash
   # Install the Celestial Backend systemd user service.
   # Symlinks the unit file, reloads systemd, enables and starts the service.
   # Run once after cloning / installing Celestial Blueprint.
   set -euo pipefail

   PROJECT="/home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint"
   UNIT_SRC="$PROJECT/systemd/celestial-backend.service"
   UNIT_DST="$HOME/.config/systemd/user/celestial-backend.service"

   mkdir -p "$(dirname "$UNIT_DST")"

   if [[ -L "$UNIT_DST" ]] && [[ "$(readlink "$UNIT_DST")" == "$UNIT_SRC" ]]; then
       echo "Symlink already in place: $UNIT_DST -> $UNIT_SRC"
   else
       ln -sf "$UNIT_SRC" "$UNIT_DST"
       echo "Linked: $UNIT_DST -> $UNIT_SRC"
   fi

   systemctl --user daemon-reload
   systemctl --user enable --now celestial-backend

   echo ""
   echo "Service status:"
   systemctl --user status celestial-backend --no-pager -l || true
   echo ""
   echo "Logs: journalctl --user -u celestial-backend -f"
   ```

4. Make it executable:
   ```bash
   chmod +x /home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/link_service.sh
   ```
