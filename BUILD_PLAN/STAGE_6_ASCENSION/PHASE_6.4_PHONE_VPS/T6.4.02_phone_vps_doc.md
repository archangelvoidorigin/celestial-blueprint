   # Phone-as-VPS Deployment

   Run the Celestial backend on an Android phone (via Termux or a Linux environment) or any ARM64 server, accessible from your desktop via Tailscale.

   ## Requirements

   - Android phone with [Termux](https://termux.dev) + Docker (via `pkg install docker`), **or** a dedicated ARM Linux VPS
   - [Tailscale](https://tailscale.com) installed on both phone and desktop
   - Docker Buildx with arm64 support on your build machine (Arch: `pacman -S docker-buildx`)

   ## Step 1 — Build the image

   On your Arch desktop (build machine):
   ```bash
   cd /home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint

   # Build and push to a local registry or save as tarball
   docker buildx build --platform linux/arm64 -t celestial-blueprint:arm64 --load .
   docker save celestial-blueprint:arm64 | gzip > celestial-arm64.tar.gz
   ```

   ## Step 2 — Transfer to phone

   ```bash
   # Via scp over Tailscale (replace with your phone's Tailscale IP)
   scp celestial-arm64.tar.gz user@100.x.x.x:~/
   ```

   ## Step 3 — Load and run on phone

   On the phone (Termux or SSH):
   ```bash
   docker load < ~/celestial-arm64.tar.gz
   docker run -d \
     --name celestial \
     -p 8787:8787 \
     -v celestial_data:/data \
     -e CELESTIAL_AUTH_TOKEN=your-secret-token \
     celestial-blueprint:arm64
   ```

   ## Step 4 — Connect from desktop

   ```bash
   # Replace with your phone's Tailscale IP
   export CELESTIAL_API_URL=http://100.x.x.x:8787
   export CELESTIAL_AUTH_TOKEN=your-secret-token

   # Test connection
   curl $CELESTIAL_API_URL/api/health
   ```

   Point the frontend's `VITE_API_URL` env var (in `frontend/.env`) at the phone's Tailscale IP.

   ## Step 5 — Auto-start on phone

   In Termux, add to `~/.bashrc` (or use a Termux:Boot script):
   ```bash
   docker start celestial 2>/dev/null || true
   ```

   ## Notes

   - The Docker volume `celestial_data` persists the SQLite DB and blueprint markdown across restarts.
   - The `CELESTIAL_AUTH_TOKEN` must match between server and client. Keep it out of version control.
   - Port 8787 must be open in the phone's firewall (Termux has no firewall by default).
   - Tailscale is preferred over direct IP — handles NAT, dynamic IPs, and provides encryption.
   ```
