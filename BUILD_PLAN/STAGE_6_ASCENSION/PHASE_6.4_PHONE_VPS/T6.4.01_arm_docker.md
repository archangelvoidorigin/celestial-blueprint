# TASK T6.4.01 — `Dockerfile` (multi-arch) + `docker-compose.arm.yml`

- **Stage / Phase**: Stage 6 — Ascension / Phase 6.4
- **Depends on**: T1.8.01
- **Produces**: `Dockerfile`, `docker-compose.arm.yml`, `.dockerignore`

## Context

`python:3.11-slim` is a multi-arch manifest image — it has both amd64 and arm64 variants. Building without `--platform` uses the host arch; `docker buildx build --platform linux/arm64,linux/amd64` produces a manifest for both. The Dockerfile is a two-stage build: a builder stage installs Python deps, a runtime stage copies only the venv + app. Data (SQLite DB, blueprints dir) is mounted as volumes so it persists across container restarts. `docker-compose.arm.yml` is identical to the base compose file (from T1.7.04) except it references a named volume and sets `platform: linux/arm64` explicitly for ARM-only deployment. This file is a companion, not a replacement — the default `docker-compose.yml` targets amd64.

## Objective

Create a multi-arch `Dockerfile`, `docker-compose.arm.yml`, and `.dockerignore` for the Celestial backend.

## Instructions

1. Create `Dockerfile` at the project root with exactly:
   ```dockerfile
   # syntax=docker/dockerfile:1
   # Multi-arch Celestial Blueprint backend.
   # Build: docker buildx build --platform linux/arm64,linux/amd64 -t celestial-blueprint .
   # Run:   docker compose up   (or docker compose -f docker-compose.arm.yml up)

   ### Stage 1: build deps ###
   FROM python:3.11-slim AS builder
   WORKDIR /build
   COPY backend/requirements.txt .
   RUN pip install --no-cache-dir --prefix=/install -r requirements.txt

   ### Stage 2: runtime ###
   FROM python:3.11-slim AS runtime
   WORKDIR /app

   # Copy installed packages from builder
   COPY --from=builder /install /usr/local

   # Copy app source (includes app/models/schema.sql)
   COPY backend/app ./app

   # Data dirs (override with volumes in compose)
   RUN mkdir -p /data/blueprints /data/chronicles

   ENV CELESTIAL_DB_PATH=/data/celestial.db \
       CELESTIAL_BLUEPRINTS_DIR=/data/blueprints \
       CELESTIAL_CHRONICLES_DIR=/data/chronicles \
       CELESTIAL_PORT=8787

   EXPOSE 8787

   CMD ["python", "-m", "uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8787"]
   ```

2. Create `docker-compose.arm.yml` with exactly:
   ```yaml
   # Celestial Blueprint — ARM64 deployment (phone-as-VPS)
   # Usage: docker compose -f docker-compose.arm.yml up -d
   version: "3.9"

   services:
     backend:
       build:
         context: .
         dockerfile: Dockerfile
         platforms:
           - linux/arm64
       platform: linux/arm64
       ports:
         - "8787:8787"
       volumes:
         - celestial_data:/data
       env_file:
         - backend/.env
       restart: unless-stopped

   volumes:
     celestial_data:
       driver: local
   ```

3. Create `.dockerignore` at the project root with exactly:
   ```
   backend/.venv
   backend/__pycache__
   backend/**/__pycache__
   backend/celestial.db
   frontend/node_modules
   frontend/dist
   .git
   BUILD_PLAN
   docs
   hypr
   skill
   tools
   mcp
   cli
   ```
