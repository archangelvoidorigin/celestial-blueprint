# TASK T6.2.01 — Hyprland windowrules + toggle keybind (append to `hypr/celestial.conf`)

- **Stage / Phase**: Stage 6 — Ascension / Phase 6.2
- **Depends on**: T6.1.01
- **Produces**: edit to `hypr/celestial.conf`

## Context

The keybind from T6.1.01 launches `alacritty --class celestial-plan`. Without windowrules, that terminal opens at the default position and size. This task appends Hyprland v2 `windowrulev2` directives to `hypr/celestial.conf` so the window is always floating, centred, and sized 900×600, assigned to the `special:celestial` scratchpad workspace. A toggle keybind (`$mainMod + C`) is also added. All additions are APPENDED to the existing file — do not rewrite it.

`windowrulev2` syntax (Hyprland ≥ 0.35): `windowrulev2 = rule, class:(regex)`.

## Objective

Append windowrule directives and a toggle keybind to `hypr/celestial.conf`.

## Instructions

1. APPEND the following block to `hypr/celestial.conf` (use `>>`, not `>`):
   ```ini

   # --- Window rules for celestial-plan terminal ---
   windowrulev2 = float, class:(celestial-plan)
   windowrulev2 = size 900 600, class:(celestial-plan)
   windowrulev2 = center, class:(celestial-plan)
   windowrulev2 = workspace special:celestial, class:(celestial-plan)

   # $mainMod + C — toggle the Celestial scratchpad workspace
   bind = $mainMod, C, togglespecialworkspace, celestial
   ```

   Exact append command (each line a separate quoted arg — robust to copy-paste
   indentation, unlike a heredoc whose terminator must sit at column 0):
   ```bash
   printf '%s\n' \
     '' \
     '# --- Window rules for celestial-plan terminal ---' \
     'windowrulev2 = float, class:(celestial-plan)' \
     'windowrulev2 = size 900 600, class:(celestial-plan)' \
     'windowrulev2 = center, class:(celestial-plan)' \
     'windowrulev2 = workspace special:celestial, class:(celestial-plan)' \
     '' \
     '# $mainMod + C — toggle the Celestial scratchpad workspace' \
     'bind = $mainMod, C, togglespecialworkspace, celestial' \
     >> /home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/hypr/celestial.conf
   ```

2. After editing, reload Hyprland to activate the rules:
   ```bash
   hyprctl reload
   ```
   (This is a manual step — the verify command only checks the file, not the live session.)
