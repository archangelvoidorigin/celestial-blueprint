- **Stage / Phase**: Stage 4 — Nerves / Phase 4.1
- **Depends on**: T1.8.01 (Stage 1 complete — backend API built and verified). Stage 4 does **not** depend on Stage 2 or Stage 3.
- **Produces**: `mcp/__init__.py`, `mcp/requirements.txt`

## Context

The Celestial Blueprint MCP server lives at `Celestial_Blueprint/mcp/` — at the project root, NOT inside `backend/`. It is a standalone Python script that communicates with the running backend API over HTTP. The `mcp/` directory needs a package marker (`__init__.py`) and a `requirements.txt` listing its one runtime dependency (`httpx`). The script will run under the **backend venv** (`backend/.venv/bin/python3`), which already has `httpx` installed, so no separate venv is required — `requirements.txt` is a reference doc, not used for an extra install.

## Objective

Create the `mcp/` scaffold: package marker and requirements file.

## Instructions

1. Create `mcp/__init__.py` with exactly:
   ```python
   """Celestial Blueprint MCP server — exposes the REST API as an MCP tool surface."""
   ```

2. Create `mcp/requirements.txt` with exactly:
   ```
   # Runtime dependency for mcp/server.py.
   # Install into the backend venv (already done in T3.1.01):
   #   pip install -r mcp/requirements.txt
   # Or use the backend venv directly:
   #   backend/.venv/bin/pip install httpx>=0.27,<0.28
   httpx>=0.27,<0.28
   ```

## Acceptance Criteria

- [ ] `mcp/__init__.py` exists with the one-line docstring
- [ ] `mcp/requirements.txt` exists and contains `httpx>=0.27,<0.28`
