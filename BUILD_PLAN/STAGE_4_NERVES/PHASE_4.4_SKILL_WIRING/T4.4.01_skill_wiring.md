   ```markdown
   |------|-------------|
   | `blueprint_list` | Show the agent all current blueprints |
   | `blueprint_get` | Retrieve a specific blueprint by ID |
   | `blueprint_create` | Start a new planning session (outputs blueprint ID) |
   | `blueprint_patch` | Update status, version, or summary |
   | `blueprint_archive` | Retire a blueprint (flips status + moves file; never deletes) |
   | `node_add` | Add any typed node (text, metric, assumption, decision, etc.) |
   | `edge_add` | Connect two nodes with a directed relationship |
   | `assumption_add` | Record a named assumption with optional risk + status |
   | `decision_add` | Record an ADR-style decision with date |
   | `risk_add` | Log a risk with likelihood, impact, mitigation |
   | `chronicle_add` | Log a session chronicle entry (kind: raw or cycle) |
   | `blueprint_export` | Dump the full blueprint as markdown |
   | `blueprint_import` | Ingest an existing markdown blueprint |
   | `blueprint_rebuild` | Rebuild the DB index from disk (after manual file edits) |

   ### Degradation

   If the MCP server is not available, fall back to the `cb` CLI (Stage 3) or direct API calls. The data model is identical; only the transport changes.
   ```
