# TASK T4.4.02 — Install the celestial-blueprint skill into Hermes

- **Stage / Phase**: Stage 4 — Nerves / Phase 4.4
- **Depends on**: T4.4.01, T2.1.06
- **Produces**: `skill/celestial-blueprint/hermes/SKILL.md`, `skill/celestial-blueprint/link_hermes_skill.sh`

## Context

Hermes is the father of the agent lineage and must be a first-class *client* of the planner, not just a cron runner. T2.1.06 already detects Hermes; the MCP server (T4.2.01) is registered (T4.3.01); the Claude-style skill exists (T1.5.01–T1.5.06) and its MCP-tool section is finalized (T4.4.01). This task makes Hermes able to *load* the methodology as one of its own skills.

Hermes uses a different skill format than Claude Code. Hermes skills live at `~/.hermes/skills/<category>/<skill-name>/SKILL.md` and require Hermes-specific frontmatter (`name`, `description`, `version`, `platforms`, `metadata.hermes.tags`, `related_skills`) — the Claude-style `SKILL.md` frontmatter (`name` + `description` only) is **not** sufficient, so a plain symlink of the existing skill will not register correctly. We therefore keep a Hermes-format manifest inside the project (`skill/celestial-blueprint/hermes/SKILL.md`) and symlink that directory into Hermes (per the project rule: real files stay in CHAOS_DAO_FORGE; external dirs hold only symlinks). The active Hermes profile is `cdf-prototype`; the installer links into both the global skills dir and the active profile's skills dir when present.

## Objective

Create a Hermes-format skill manifest and an idempotent installer that symlinks it into Hermes so Hermes loads the celestial-blueprint planning methodology and drives the planner via its MCP tools or the `cb` CLI.

## Instructions

1. Create `skill/celestial-blueprint/hermes/SKILL.md` with exactly:
   ```markdown
   ---
   name: celestial-blueprint
   description: "Tiered, interrogation-driven planning — create and evolve structured blueprints via the Celestial MCP tools or the cb CLI."
   version: 1.0.0
   platforms: [linux]
   metadata:
     hermes:
       tags: [planning, blueprint, strategy, mcp]
       related_skills: []
   ---

   # Celestial Blueprint — Planning (Hermes)

   Turn a vague intent into a rigorous, falsifiable plan. Interrogate before you
   plan, pick the smallest tier that fits, and record assumptions / decisions /
   risks as you go.

   ## Tiers

   | Tier | When | Min iterations |
   |------|------|----------------|
   | small | one idea/decision, quick gut-check | 3 |
   | medium | a feature/project with several facets | 2 |
   | large | a whole product/strategy | 1 |
   | continuous | a living plan that evolves | ∞ |

   Start one tier lower and upgrade — cheaper to grow than to over-plan.

   ## Driving the planner

   Prefer the `celestial-blueprint` MCP tools when the server is active
   (`mcp-lazy list` shows it):

   `blueprint_create`, `blueprint_list`, `blueprint_get`, `blueprint_patch`,
   `blueprint_archive`, `node_add`, `edge_add`, `assumption_add`,
   `decision_add`, `risk_add`, `chronicle_add`, `blueprint_export`,
   `blueprint_import`, `blueprint_rebuild`.

   Typical flow: `blueprint_create` (sets tier) → interrogate the user →
   `node_add` for each finding → `assumption_add` / `decision_add` / `risk_add`
   → `chronicle_add` to log the session.

   ## Degradation

   If the MCP server is not available, use the `cb` CLI (Stage 3) or direct
   REST calls to the backend (`CELESTIAL_API_URL`, default `http://localhost:8787`).
   The data model is identical; only the transport changes. The canonical format
   is `docs/FORMAT.md`; markdown is the source of truth, the DB is a rebuildable index.
   ```

2. Create `skill/celestial-blueprint/link_hermes_skill.sh` with exactly:
   ```bash
   #!/usr/bin/env bash
   # Symlink the Hermes-format celestial-blueprint skill into Hermes' skills dirs.
   # Run from any directory; idempotent. Source stays in the project.
   set -euo pipefail

   HERE="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
   SRC="$HERE/hermes"                     # the Hermes-format skill directory
   CATEGORY="project-management"
   SKILL_NAME="celestial-blueprint"
   HERMES="$HOME/.hermes"

   link_into() {
       local skills_root="$1"
       if [ ! -d "$skills_root" ]; then
           echo "  SKIP $skills_root (absent)"
           return
       fi
       local target_dir="$skills_root/$CATEGORY"
       mkdir -p "$target_dir"
       local dest="$target_dir/$SKILL_NAME"
       if [ -L "$dest" ]; then
           echo "  EXISTS $dest"
       elif [ -e "$dest" ]; then
           echo "  SKIP $dest (non-symlink exists — remove manually)"
       else
           ln -s "$SRC" "$dest"
           echo "  LINKED $dest"
       fi
   }

   echo "Installing celestial-blueprint into Hermes..."
   link_into "$HERMES/skills"
   if [ -f "$HERMES/active_profile" ]; then
       profile="$(cat "$HERMES/active_profile")"
       [ -n "$profile" ] && link_into "$HERMES/profiles/$profile/skills"
   fi
   echo "Done. Restart Hermes (or refresh its skill index) to load it."
   ```

