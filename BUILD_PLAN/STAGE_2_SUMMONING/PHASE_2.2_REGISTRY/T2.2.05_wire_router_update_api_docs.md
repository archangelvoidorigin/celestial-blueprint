# TASK T2.2.05 — Wire agents router into main.py + update API.md

- **Stage / Phase**: Stage 2 — Summoning / Phase 2.2
- **Depends on**: T2.2.04
- **Produces**: `backend/app/main.py` (lifespan + one include edited), `docs/API.md` (agents section appended)

## Context

`backend/app/main.py` (T1.4.01) initialises the core DB in its lifespan (`init_db()`) and
includes all API routers under a `# --- ROUTER INCLUDES ---` block, each with
`dependencies=[Depends(require_auth)]` (added by T1.7.02). Stage 2 added a separate
`init_agents()` (T2.2.01) for the agents table and an agents router (T2.2.04). This task wires
both into `main.py`: call `init_agents()` in the lifespan, and include the agents router with
the same global auth dependency as the other routers. Then document the endpoints in
`docs/API.md` (T1.4.07).

## Objective

Initialise the agents table on startup and register the agents router (auth-gated) in
`main.py`, then document the six new endpoints in `docs/API.md`.

## Instructions

### 1. Update `backend/app/main.py`

a. Change the db import to also import `init_agents`:
```python
from app.core.db import init_db, init_agents
```

b. In the `lifespan` function, call `init_agents()` right after `init_db()`:
```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    init_db()
    init_agents()
    yield
```

c. In the `# --- ROUTER INCLUDES ---` block, after the existing `app.include_router(...)`
   lines, add the agents router with the same auth dependency the other routers use
   (`Depends` and `require_auth` are already imported by T1.7.02):
```python
from app.api.agents import router as agents_router
app.include_router(agents_router, dependencies=[Depends(require_auth)])
```

### 2. Append to `docs/API.md`

Add the following section at the end of the file:

```markdown
## Agents

### GET /api/agents
List all registered agents.
Query params: `status` (optional) — filter by `active`, `inactive`, or `unknown`.
Returns: `array<Agent>`

### GET /api/agents/{id}
Get a single agent by id.
Returns: `Agent` or 404.

### POST /api/agents
Manually register or update an agent.
Body: `AgentCreate` — `{id, name, capabilities[], activation_cmd, status, version, models[]}`
Returns: `Agent` (201 Created).

### PATCH /api/agents/{id}
Partially update an agent (any subset of fields).
Body: `AgentPatch` — all fields optional.
Returns: `Agent` or 404.

### DELETE /api/agents/{id}
Remove an agent from the registry.
Returns: 204 No Content or 404.

### POST /api/agents/scan
Trigger a full system scan for agent runtimes.
Upserts all discovered agents into the DB.
Returns: updated `array<Agent>`.
```
