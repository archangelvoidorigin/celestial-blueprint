patch fields use `Optional[...] = None`. No `ConfigDict` — Stage 1 does not use it and these

## Objective

Append three Pydantic models to `backend/app/models/schemas.py` for the agent entity.

## Instructions

2. Append at the end of the file:

```python
# ── Agent Registry ────────────────────────────────────────────────────────────

class AgentCreate(BaseModel):
    id: str
    name: str
    capabilities: list[str] = Field(default_factory=list)
    activation_cmd: str = ""
    status: str = "unknown"
    version: str = ""
    models: list[str] = Field(default_factory=list)


class Agent(AgentCreate):
    discovered_at: str = ""
    updated_at: str = ""


class AgentPatch(BaseModel):
    name: Optional[str] = None
    capabilities: Optional[list[str]] = None
    activation_cmd: Optional[str] = None
    status: Optional[str] = None
    version: Optional[str] = None
    models: Optional[list[str]] = None
```

   No import changes are needed: `BaseModel`, `Field`, and `Optional` are already imported at
   the top of `schemas.py` from Stage 1 (T1.3.03). Do not add a `ConfigDict` import.
