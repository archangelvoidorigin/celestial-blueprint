## Context

Capability profiles define what each agent can do using a controlled vocabulary of 8 terms
(see Phase 2.3 README). The static map is the fallback — detectors may override it. Having
it in a single module means the routing layer (Phase 2.4) can query capabilities without
re-reading the DB. It also provides `CAPABILITY_VOCAB` as the authoritative list of valid
capability strings.

## Objective

Create `backend/agents/capability_map.py` with `CAPABILITY_VOCAB`, `DEFAULT_CAPABILITIES`,
and `get_capabilities(agent_id)`.

## Instructions

Create `backend/agents/capability_map.py` with this content:

```python
"""
Celestial Blueprint — Capability Map
Static mapping from agent id → capability list.
Detectors may override; routing layer uses this as fallback.
"""
from __future__ import annotations

CAPABILITY_VOCAB: frozenset[str] = frozenset({
    "research",
    "code_gen",
    "critique",
    "summarise",
    "plan",
    "file_ops",
    "web_search",
    "mcp_tools",
})

# Default capability set per agent id.
# Keys match the `id` field in AgentCreate / agent_entry().
DEFAULT_CAPABILITIES: dict[str, list[str]] = {
    "claude-code": ["code_gen", "file_ops", "mcp_tools", "plan", "critique"],
    "opencode":    ["code_gen", "file_ops", "plan", "mcp_tools"],
    "codex":       ["code_gen", "file_ops", "web_search"],
    "antigravity": ["code_gen", "file_ops", "web_search", "plan", "mcp_tools"],
    "hermes":      ["plan", "file_ops", "code_gen", "summarise"],
    "ollama":      ["code_gen", "summarise", "research", "critique"],
}

# Fallback for unknown agents
_FALLBACK: list[str] = ["code_gen"]


def get_capabilities(agent_id: str) -> list[str]:
    """Return the canonical capability list for an agent id."""
    return list(DEFAULT_CAPABILITIES.get(agent_id, _FALLBACK))


def validate_capabilities(caps: list[str]) -> list[str]:
    """Return only vocab-valid capability strings; drop unknowns silently."""
    return [c for c in caps if c in CAPABILITY_VOCAB]
```
