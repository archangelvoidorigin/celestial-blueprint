# TASK T2.4.03 — Wire route router into main.py + document in API.md

- **Stage / Phase**: Stage 2 — Summoning / Phase 2.4
- **Depends on**: T2.4.02
- **Produces**: `backend/app/main.py` (one line added), `docs/API.md` (route section appended)

## Context

Same pattern as T2.2.05: add one `include_router` call to `app/main.py` for the route
router, then document the endpoint in `docs/API.md`. The router lives at
`app.api.router` — note this is different from `app.api.agents` added in T2.2.05.

## Objective

Register the route router in `main.py` and append the `/api/route` documentation to
`docs/API.md`.

## Instructions

### 1. Update `backend/app/main.py`

Find the `# --- ROUTER INCLUDES ---` block and add after the agents router line, applying the
same global auth dependency the other routers use (`Depends` and `require_auth` are already
imported by T1.7.02):

```python
from app.api.router import router as route_router
app.include_router(route_router, dependencies=[Depends(require_auth)])
```

### 2. Append to `docs/API.md`

```markdown
## Routing

### POST /api/route
Resolve the best available agent for a task type.

Body:
```json
{
  "task_type": "code_gen",
  "context": "optional additional context string"
}
```

`task_type` must be one of the capability vocabulary values:
`research`, `code_gen`, `critique`, `summarise`, `plan`, `file_ops`, `web_search`, `mcp_tools`.

Returns:
```json
{
  "agent_id": "claude-code",
  "agent_name": "Claude Code",
  "activation_cmd": "/usr/bin/claude",
  "task_type": "code_gen",
  "reason": "highest-priority active agent with capability 'code_gen'"
}
```
```
