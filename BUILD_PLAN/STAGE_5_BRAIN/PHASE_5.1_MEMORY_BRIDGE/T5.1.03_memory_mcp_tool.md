## Context

The MCP server (T4.2.01) is the primary agent interface to Celestial Blueprint. Adding a `memory_query` tool allows agents to ask the backend to fetch brain context for a blueprint without leaving the MCP session. The backend's `/api/blueprints/{id}/memory-context` endpoint already exists (T5.1.02) and degrades gracefully — the MCP tool inherits that behaviour. The existing `_get` helper is used; a `?topic=` param is passed inline. The tool count goes from 14 → 15. **Note:** Stage 4's `VERIFY.md` asserts exactly 14 tools — that assertion is correct *before* this task and is intentionally superseded here. Do not re-run the Stage 4 14-tool check after applying T5.1.03; the Stage 5 `VERIFY.md` checks for the `memory_query` tool by name rather than by count.

## Objective

Add the `memory_query` tool descriptor and dispatch branch to `mcp/server.py` and update all tool-count references in the file.
