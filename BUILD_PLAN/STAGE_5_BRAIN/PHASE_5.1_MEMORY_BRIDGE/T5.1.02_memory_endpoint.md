# TASK T5.1.02 — `GET /api/blueprints/{id}/memory-context` endpoint

- **Stage / Phase**: Stage 5 — Brain / Phase 5.1
- **Depends on**: T5.1.01
- **Produces**: `backend/app/api/memory.py`, edit to `backend/app/main.py`, edit to `docs/API.md`

## Context

`query_memory` from T5.1.01 can fetch context for any topic, but the backend needs an HTTP endpoint so the frontend, CLI, and MCP server can request it without importing Python services directly. This endpoint looks up the blueprint name (or accepts an optional `?topic=` override) and calls `query_memory`. It is part of the blueprint router family and follows the same 404/graceful pattern as the other blueprint endpoints.

## Objective

Create `app/api/memory.py` with `GET /api/blueprints/{id}/memory-context` and register it in `main.py`.

## Instructions

1. Create `backend/app/api/memory.py` with exactly:
   ```python
   """Memory-context endpoint: surface brain snippets for a blueprint."""
   from __future__ import annotations

   from fastapi import APIRouter, HTTPException

   from app.services import store
   from app.services.memory import query_memory

   router = APIRouter(prefix="/api/blueprints", tags=["memory"])


   @router.get("/{bid}/memory-context")
   def get_memory_context(bid: str, topic: str = "", limit: int = 5):
       bp = store.get_blueprint(bid)
       if bp is None:
           raise HTTPException(status_code=404, detail="blueprint not found")
       q = topic or bp["frontmatter"].get("name", bid)
       results = query_memory(q, limit=limit)
       return {"blueprint_id": bid, "topic": q, "results": results}
   ```

2. Edit `backend/app/main.py`: under `# --- ROUTER INCLUDES ---`, ADD:
   ```python
   from app.api.memory import router as memory_router
   app.include_router(memory_router)
   ```

3. Edit `docs/API.md`: add under a `## Stage 5 — Brain` section (append at end):
   ```markdown
   ## Stage 5 — Brain

   ### `GET /api/blueprints/{id}/memory-context`
   Query the brain-orchestrator for context snippets relevant to the blueprint's topic.
   - Query params: `topic` (string, optional — defaults to blueprint name), `limit` (int, default 5)
   - Returns: `{"blueprint_id": str, "topic": str, "results": [{"text": str, "type": str}]}`
   - Returns `[]` results if `CELESTIAL_BRAIN_ORCHESTRATOR_PATH` is not set (graceful degradation).
   ```
