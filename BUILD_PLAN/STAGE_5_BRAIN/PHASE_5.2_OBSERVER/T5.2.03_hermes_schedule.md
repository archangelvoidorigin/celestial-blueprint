- **Stage / Phase**: Stage 5 — Brain / Phase 5.2

## Context

Hermes is the CHAOS_DAO_FORGE task scheduler (`~/.hermes/` or installed via pip). It runs YAML-defined tasks on schedule. The observer script (T5.2.01) should run nightly so any blueprint that has gone stale since the last planning session gets a drift assumption logged before the next session starts. The Hermes task file goes in `tools/` alongside the observer. `docs/automation.md` is the automation reference doc (Stage 3, T3.4.01).

## Objective

Create the Hermes task YAML for nightly observer runs and document it in `docs/automation.md`.

## Instructions

1. Create `tools/hermes_observer_task.yaml` with exactly:
   ```yaml
   # Celestial Blueprint — nightly observer task
   # Install: hermes task add tools/hermes_observer_task.yaml
   name: celestial-observer
   description: "Detect stale blueprints and log drift assumptions"
   schedule: "0 2 * * *"   # 02:00 UTC nightly
   working_dir: "/home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint"
   command:
     - "/home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/backend/.venv/bin/python3"
     - "tools/observer.py"
     - "--stale-days"
     - "3"
   env:
     CELESTIAL_API_URL: "http://localhost:8787"
     CELESTIAL_AUTH_TOKEN: "${CELESTIAL_AUTH_TOKEN}"
   on_failure: log
   ```
