# TASK T5.5.02 — Wire archiver into `POST /api/blueprints/{id}/archive`

- **Stage / Phase**: Stage 5 — Brain / Phase 5.5
- **Depends on**: T5.5.01
- **Produces**: edit to `backend/app/api/blueprints.py`

## Context

The `archive_blueprint` endpoint in `app/api/blueprints.py` (T1.4.02) already calls `store.archive_blueprint(bid)` (sets status to `archived`) and `write_blueprint_file(bid)` (rewrites the markdown). Stage 5 extends this with a call to `archive_blueprint_file(bid)` AFTER `write_blueprint_file` — so the file is first rewritten with the current DB state (ensuring the archived markdown is complete), then moved to the archive directory. The endpoint return value gains an `"archive_path"` key (or `null` if not configured).

## Objective

Edit `archive_blueprint` in `app/api/blueprints.py` to call `archive_blueprint_file` and include the result in the response.

## Instructions

1. Edit `backend/app/api/blueprints.py`:

   a. Add import (after existing imports):
      ```python
      from app.services.archiver import archive_blueprint_file
      ```

   b. Replace the `archive_blueprint` function with:
      ```python
      @router.post("/{bid}/archive")
      def archive_blueprint(bid: str):
          if not store.archive_blueprint(bid):
              raise HTTPException(status_code=404, detail="blueprint not found")
          write_blueprint_file(bid)
          archive_path = archive_blueprint_file(bid)
          return {"status": "archived", "id": bid, "archive_path": archive_path}
      ```

## Acceptance Criteria

- [ ] `archive_blueprint` imports `archive_blueprint_file`
- [ ] `POST /api/blueprints/{id}/archive` returns `{"status":"archived","id":...,"archive_path":null}` when `CELESTIAL_ARCHIVE_DIR` is not set
- [ ] When `CELESTIAL_ARCHIVE_DIR` is set, blueprint file is moved and `archive_path` is non-null
- [ ] Returns 404 for unknown blueprint id
