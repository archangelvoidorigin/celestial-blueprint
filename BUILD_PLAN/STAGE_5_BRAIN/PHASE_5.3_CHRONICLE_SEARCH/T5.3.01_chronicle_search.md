# TASK T5.3.01 — `GET /api/chronicles/search?q=` cross-blueprint search

- **Stage / Phase**: Stage 5 — Brain / Phase 5.3
- **Depends on**: T1.8.01
- **Produces**: edit to `backend/app/services/store.py`, edit to `backend/app/api/chronicles.py`, edit to `backend/app/main.py` (if separate router), edit to `docs/API.md`

## Context

Chronicles are per-blueprint session journals stored in the `chronicles` SQLite table (T1.3.01). A planning session often needs to search past chronicles across ALL blueprints — e.g. "what did I learn last time I planned something in this realm?". This task adds `store.search_chronicles(q, limit)` (LIKE query on `body`) and wires it to a new `GET /api/chronicles/search?q=` endpoint. The endpoint is registered at the top-level `/api/chronicles` prefix (not under `/api/blueprints/`), so it needs either a new router or a new prefix on the existing chronicles router — use a new file to avoid ambiguity.

## Objective

Add `search_chronicles` to the store and expose `GET /api/chronicles/search?q=` as a new endpoint.

## Instructions

1. Edit `backend/app/services/store.py`: add the `search_chronicles` function at the end of the file:
   ```python
   def search_chronicles(q: str, limit: int = 50) -> list[dict]:
       """Search chronicle bodies across all blueprints using LIKE."""
       with connection() as c:
           rows = c.execute(
               """SELECT id, blueprint_id, kind, trigger, cycle,
                         body, path, created_at
                  FROM chronicles
                  WHERE body LIKE ?
                  ORDER BY created_at DESC
                  LIMIT ?""",
               (f"%{q}%", limit),
           ).fetchall()
       return [dict(r) for r in rows]
   ```

2. Create `backend/app/api/chronicles_search.py` with exactly:
   ```python
   """Cross-blueprint chronicle search."""
   from __future__ import annotations

   from fastapi import APIRouter

   from app.services import store

   router = APIRouter(prefix="/api/chronicles", tags=["chronicles"])


   @router.get("/search")
   def search_chronicles(q: str = "", limit: int = 50):
       if not q:
           return []
       return store.search_chronicles(q, limit=limit)
   ```

3. Edit `backend/app/main.py`: under `# --- ROUTER INCLUDES ---`, ADD:
   ```python
   from app.api.chronicles_search import router as chronicles_search_router
   app.include_router(chronicles_search_router)
   ```

   **Important**: register `chronicles_search_router` BEFORE the existing `chronicles_router` (which uses `/{bid}/chronicles`), or ensure the `/search` path is declared before `/{bid}` in FastAPI's route order. Since these are different routers with different prefixes (`/api/chronicles` vs `/api/blueprints`), there is no conflict.

4. Edit `docs/API.md`: under `## Stage 5 — Brain`, append:
   ```markdown
   ### `GET /api/chronicles/search`
   Full-text search across all blueprints' chronicle bodies.
   - Query params: `q` (string, required), `limit` (int, default 50)
   - Returns: array of `{id, blueprint_id, kind, trigger, cycle, body, path, created_at}`
   - Returns `[]` for empty `q`.
   ```
