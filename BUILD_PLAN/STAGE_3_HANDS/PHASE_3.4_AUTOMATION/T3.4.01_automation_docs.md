## Context

The `cb` CLI is designed to be called by automated agents and shell scripts, not just humans. The most common automation needs are: (1) Hermes agent tasks that call `cb rebuild` nightly to keep the database in sync with the markdown blueprints directory, and (2) cron jobs for environments where Hermes is not running. `docs/automation.md` documents both patterns plus a fish alias shortcut.

The `docs/` directory lives at `Celestial_Blueprint/docs/` and already contains `FORMAT.md` and `API.md` (created in earlier stages).

   # CLI Automation

   The `cb` CLI is a thin shell over the REST API — no server needed beyond the running backend. Any tool that can run a shell command can automate it.

   ## Environment variables

   Set these before any automated invocation:

   ```bash
   export CELESTIAL_API_URL=http://localhost:8787
   export CELESTIAL_AUTH_TOKEN=your-token-here   # omit if auth gate is open
   ```

   Or put them in `~/.config/celestial/config.toml`:

   ```toml
   [default]
   api_url = "http://localhost:8787"
   token   = "your-token-here"
   ```

   ---

   ## Hermes automation

   Hermes reads task definitions from YAML files. The example below runs `cb rebuild` every night at 02:00, then logs the result to a chronicle.

   ### Sample Hermes task YAML

   Save this to `~/.hermes/tasks/celestial-nightly-rebuild.yaml` (or the Hermes tasks directory configured in your Hermes installation):

   ```yaml
   name: celestial-nightly-rebuild
   description: Rebuild the Celestial Blueprint DB from markdown files and log the result
   schedule: "0 2 * * *"   # cron syntax — 02:00 every night
   env:
     CELESTIAL_API_URL: "http://localhost:8787"
   steps:
     - name: rebuild
       run: cb rebuild --pretty
       capture: result

     - name: log-chronicle
       run: |
         cb chronicle add __system__ \
           --kind raw \
           --trigger "nightly-rebuild" \
           --body "Nightly rebuild completed. Result: $(echo $result | python3 -c 'import sys,json; d=json.load(sys.stdin); print(d[\"imported\"])') blueprints imported."
       ignore_errors: true
   ```

   > **Note:** `__system__` is a placeholder blueprint ID. Replace it with the ID of a dedicated "System Logs" blueprint, or remove the `log-chronicle` step if you don't need chronicle tracking.

   ### Running a Hermes task manually

   ```bash
   hermes run celestial-nightly-rebuild
   ```

   ### One-shot rebuild via Hermes

   ```bash
   hermes exec "cb rebuild --pretty"
   ```

   ---

   ## Cron automation

   For environments without Hermes, use the system cron daemon directly.

   ### Add to crontab

   Run `crontab -e` and add:

   ```cron
   # Celestial Blueprint — nightly DB rebuild at 02:00
   0 2 * * * CELESTIAL_API_URL=http://localhost:8787 /path/to/.venv/bin/cb rebuild >> /tmp/celestial-rebuild.log 2>&1
   ```

   Replace `/path/to/.venv/bin/cb` with the absolute path to the `cb` binary in your venv. Find it with:

   ```bash
   which cb
   ```

   ### Verify cron output

   ```bash
   tail -f /tmp/celestial-rebuild.log
   ```

   ---

   ## Fish shell aliases

   Add to `~/.config/fish/config.fish` or `~/.config/fish/conf.d/celestial.fish`:

   ```fish
   # Short alias for cb
   alias bp='cb'

   # Quick blueprint list
   alias bpl='cb blueprint list --pretty'

   # Quick create (usage: bpnew "My Plan" small)
   function bpnew
       cb blueprint create $argv[1] --tier $argv[2]
   end

   # Export to clipboard (requires wl-copy on Wayland or xclip on X11)
   function bpexport
       cb blueprint export $argv[1] | wl-copy
       echo "Exported blueprint $argv[1] to clipboard"
   end
   ```

   ---

   ## Agent-friendly JSON output

   All `cb` commands except `blueprint export` return JSON. Use `--field` to extract specific values in shell scripts:

   ```bash
   # Get just the blueprint ID from a create call
   BP_ID=$(cb blueprint create "CI Smoke Test" --tier small --field id 2>/dev/null \
     | tr -d '"')

   # Add a node using the captured ID
   cb node add "$BP_ID" --type text --title "Smoke node"
   ```

   Use `--pretty` for human-readable output in logs:

   ```bash
   cb blueprint list --pretty
   ```

   ---

   ## Systemd timer (alternative to cron)

   If you prefer systemd over cron:

   **`~/.config/systemd/user/celestial-rebuild.service`:**
   ```ini
   [Unit]
   Description=Celestial Blueprint nightly DB rebuild

   [Service]
   Type=oneshot
   Environment=CELESTIAL_API_URL=http://localhost:8787
   ExecStart=/path/to/.venv/bin/cb rebuild
   StandardOutput=journal
   ```

   **`~/.config/systemd/user/celestial-rebuild.timer`:**
   ```ini
   [Unit]
   Description=Run Celestial Blueprint rebuild nightly

   [Timer]
   OnCalendar=*-*-* 02:00:00
   Persistent=true

   [Install]
   WantedBy=timers.target
   ```

   Enable:
   ```bash
   systemctl --user enable --now celestial-rebuild.timer
   systemctl --user status celestial-rebuild.timer
   ```
