# TASK T3.3.01 — `cli/completions/cb.fish` — fish shell completions

- **Stage / Phase**: Stage 3 — Hands / Phase 3.3
- **Depends on**: T3.1.04
- **Produces**: `backend/cli/completions/cb.fish`

## Context

The `backend/cli/completions/` directory was created in T3.1.01 (with a `.gitkeep`). This task writes the fish completion definitions for the `cb` CLI. Fish completions use `complete -c cb` directives. Each subcommand group (`blueprint`, `node`, `edge`, `assumption`, `decision`, `risk`, `chronicle`, `rebuild`) needs its own completion entries for both the group name and the actions/flags within it. The condition `__fish_seen_subcommand_from` is used to scope flags to their parent subcommand.

## Objective

Create `backend/cli/completions/cb.fish` with complete fish tab-completion coverage for all `cb` commands and flags.

## Instructions
