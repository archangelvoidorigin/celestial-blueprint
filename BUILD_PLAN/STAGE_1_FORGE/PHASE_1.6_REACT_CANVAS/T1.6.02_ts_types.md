After this task, update `src/lib/api.ts` to import from here instead of defining inline stubs (replace the "local type stubs" block with `export type { ... } from "../types/blueprint"`).

## Objective

## Instructions
1. Create `frontend/src/types/blueprint.ts` with exactly:
   ```ts
   /** Canonical blueprint types — mirror of app/models/schemas.py. */

   export interface BlueprintSummary {
     id: string;
     name: string;
     version: string;
     tier: "small" | "medium" | "large" | "continuous";
     status: "draft" | "active" | "blocked" | "shipped" | "archived";
     updated_at: string;
   }

   export type NodeType =
     | "text" | "metric" | "chart" | "list" | "quote" | "research"
     | "assumption" | "decision" | "chronicle" | "agent" | "cost"
     | "realm" | "tool";

   export interface BpNode {
     id: string;
     cat: string;
     type: NodeType;
     title: string;
     body: string;
     data: Record<string, unknown>;
   }

   export interface BpEdge {
     src: string;
     dst: string;
     label: string;
   }

   export interface Assumption {
     id: string; bp_ref: string; status: string; risk: string; body: string;
   }
   export interface Decision {
     id: string; bp_ref: string; status: string; date: string; body: string;
   }
   export interface Risk {
     id: string; risk: string; likelihood: string; impact: string;
     mitigation: string; status: string;
   }
   export interface Research { id: string; query: string; date: string; body: string; }
   export interface ChronicleLink { id: string; type: string; path: string; }
   export interface Category { id: string; name: string; }
   export interface Version { version: string; date: string; note: string; }

   export interface Blueprint {
     frontmatter: Record<string, unknown>;
     summary: string;
     categories: Category[];
     nodes: BpNode[];
     assumptions: Assumption[];
     decisions: Decision[];
     risks: Risk[];
     research: Research[];
     chronicle_links: ChronicleLink[];
     edges: BpEdge[];
     versions: Version[];
   }

   export interface Chronicle {
     id: number; blueprint_id: string; kind: string; trigger: string;
     cycle: string; path: string; body: string; created_at: string;
   }
   ```
2. Edit `frontend/src/lib/api.ts`: delete everything from the comment `// ----- local type stubs` to the end of the file, and replace it with:
   ```ts
   import type {
     Blueprint, BlueprintSummary, BpNode as Node, BpEdge as Edge,
     Assumption, Decision, Risk, Research, ChronicleLink, Version, Chronicle,
   } from "../types/blueprint";

   export type {
     Blueprint, BlueprintSummary, Node, Edge,
     Assumption, Decision, Risk, Research, ChronicleLink, Version, Chronicle,
   };
   ```
   > Use `import type` (not `export type { ... } from`): api.ts's own function
   > signatures reference these names locally (`req<Blueprint>`, `Partial<Node>`,
   > `req<Chronicle[]>`, …). A bare `export … from` re-export creates **no local
   > binding**, so the build would fail with "Cannot find name 'Blueprint'". The
   > separate `export type { … }` line re-exports them for any component that
   > imports a type from `../lib/api`.
