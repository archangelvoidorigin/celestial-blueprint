## Context
Blueprints come in four tiers (small/medium/large/continuous), defined in `docs/FORMAT.md`. Each tier has a standard output shape. These template files are (1) starting points a planner copies, and (2) the parser's test fixtures in later tasks — so they MUST be valid per `docs/FORMAT.md` (correct frontmatter + HTML-comment anchors). Read `docs/FORMAT.md` before writing these; the anchors here must match its "Section Layout" exactly.

## Objective
Create four valid blueprint markdown templates, one per tier, conforming to `docs/FORMAT.md`.

## Instructions
1. Create `docs/templates/small.md` with exactly:
   ```markdown
   ---
   id: bp_2026-01-01_small-template
   name: "Small Template"
   version: 0.1.0
   tier: small
   status: draft
   realm: 00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/smart-folder.md

   pipeline_stage: ideation
   date_created: 2026-01-01
   modified: 2026-01-01
   model: haiku
   iterations: 0

   # Small Template

   <!-- bp:summary -->
   A minimal single-file blueprint for a quick 5–15 minute planning pass.

   ## Categories
   - <!-- cat:c1 --> Validation
   - <!-- cat:c2 --> Technical

   ## Nodes

   ### Core Idea
   <!-- node:text id=n1 cat=c1 -->
   One paragraph describing the core idea being planned.

   ### Key Assumption
   <!-- node:assumption id=n2 cat=c1 status=unvalidated risk=high falsifier="state what would prove this false" -->

   ### Recommendation
   <!-- node:decision id=n3 cat=c2 status=deferred supersedes=- -->

     The recommendation, pending validation.

   <!-- edges -->

   <!-- versions -->
   ```
