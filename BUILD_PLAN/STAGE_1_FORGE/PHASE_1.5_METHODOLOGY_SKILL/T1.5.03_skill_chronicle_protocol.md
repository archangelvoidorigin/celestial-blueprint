## Context
Chronicles are the session memory: short RAW entries captured at key moments, and a CYCLE summary at session end. They make planning auditable and let later sessions (and the future Brain stage) learn from history. RAW entries map to `POST /api/blueprints/{id}/chronicles` with `kind=raw`; the cycle summary uses `kind=cycle`. The seven triggers below define *when* to chronicle.

## Objective
Append the chronicle protocol (7 triggers + RAW/CYCLE formats) to `SKILL.md`.

## Instructions
1. APPEND to `skill/celestial-blueprint/SKILL.md` exactly:
   ```markdown
   ## Step 2 — Chronicle as you go

   Record the session's reasoning. Two kinds: RAW (in-the-moment) and CYCLE
   (end-of-session summary). In MCP mode, POST them to the chronicles endpoint;
   standalone, write markdown files under `chronicles/raw/` and
   `chronicles/cycles/<n>/`.

   **Write a RAW chronicle at any of these 7 triggers:**
   1. An assumption is challenged or invalidated.
   2. A decision is made (or reversed).
   3. Research changes the direction.
   4. A risk is discovered.
   5. The tier is upgraded/downgraded.
   6. The user reveals a constraint or scar that reframes the plan.
   7. You hit a dead end or contradiction.

   **RAW format** (`chronicles/raw/YYYY-MM-DD-<n>-<slug>.md`):
   ```
   # RAW — <trigger> — <date>
   **Context:** what was happening.
   **Event:** what triggered this.
   **Impact:** what changed in the plan (nodes/assumptions/decisions touched).
   ```

   **CYCLE format** (`chronicles/cycles/<n>/session-NNN.md`), written at session end:
   ```
   # CYCLE <n> — session NNN — <date>
   **Goal:** what this session set out to plan.
   **What we learned:** 3–6 bullets.
   **Decisions:** list with status.
   **Open assumptions/risks:** carried forward.
   **Next session:** the one thing to do next.
   ```

   On a CYCLE chronicle, bump the blueprint `version` (semver patch) and append
   a Version Log entry.
   ```
