## Context
The methodology skill is the behaviour spec that makes Celestial Blueprint a *planning* tool, not just a database. It is usable standalone by any agent that loads skills (Claude Code, OpenCode, etc.) and also drives the web app. This phase builds `SKILL.md` section by section. This first task creates the file with its frontmatter and the tier-selection table — how to pick small/medium/large/continuous for a planning request. Tiers and their defaults are defined in `docs/FORMAT.md`; keep them consistent.

## Objective

   ```markdown
   ---
   name: celestial-blueprint
   description: >-
     Tiered, interrogation-driven planning. Use when the user wants to plan,
     scope, or pressure-test a project, feature, or decision and capture it as
     a structured blueprint (markdown + optional canvas). Picks a planning tier,
     runs a blank-slate interrogation before planning, tracks assumptions /

   # Celestial Blueprint — Planning Methodology

   You turn a vague intent into a rigorous, falsifiable plan. You do this in
   four tiers, you interrogate before you plan, and you record what you learn.

   Canonical data format: `docs/FORMAT.md`. When the backend API is available
   (MCP mode), persist via it; otherwise write the markdown directly (see the
   degradation section).

   ## Step 0 — Select the tier

   Pick the smallest tier that fits. Confirm the tier with the user in one line
   before proceeding.

   | Tier | When | Time | Model | Min iterations | Output |
   |------|------|------|-------|----------------|--------|
   | **small** | one idea/decision, quick gut-check | 5–15 min | haiku | 3 | 5–10 nodes, 3 assumptions, 1 recommendation |
   | **medium** | a feature/project with several facets | 30–60 min | sonnet | 2 | 5 categories, research-backed nodes, risk table, decision log |
   | **large** | a whole product/strategy | 2–4 hr | opus | 1 | 40+ nodes, metrics+charts, full risk matrix, financials |
   | **continuous** | a living plan that evolves | ongoing | adaptive | ∞ | any tier, auto-appended over time |

   Selection heuristics:
   - Single yes/no or "should I…" → **small**.
   - "Plan the X feature / sprint / migration" → **medium**.
   - "Plan the company / product / Q3 strategy" → **large**.
   - "Keep a running plan for X" / revisited weekly → **continuous**.

   When unsure, start one tier *lower* and upgrade — cheaper to grow than to
   over-plan. Tier sets the frontmatter `tier`, `model`, and `iterations`.
   ```
