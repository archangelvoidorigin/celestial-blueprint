# TASK T1.8.02 — Recovery test (delete DB → rebuild from markdown)

- **Stage / Phase**: Stage 1 — Forge / Phase 1.8
- **Depends on**: T1.4.06
- **Produces**: evidence (no new files — this is a run-and-verify task)

## Context
"Markdown is canonical" means the SQLite DB is fully rebuildable from `blueprints/*.md`. This test proves it: delete the DB while the app is running, call `POST /api/rebuild`, and confirm all blueprints are back. This is the disaster-recovery guarantee. It can be run against the live dev server or via a TestClient script.

## Objective
Prove that deleting `celestial.db` and calling `POST /api/rebuild` restores all blueprints from disk.

## Instructions
1. Ensure at least one blueprint exists (create one through the UI or API if needed).
2. Confirm the blueprint file exists on disk:
   ```bash
   ls /home/void-yggdrasil/.claude/projects/-home-void-yggdrasil-CHAOS-DAO-FORGE/memory/blueprints/
   ```
   (Or wherever `CELESTIAL_BLUEPRINTS_DIR` points.)
3. Stop the backend (`Ctrl-C` in the `dev.fish` terminal, or kill the uvicorn process).
4. Delete the database:
   ```bash
   rm /home/void-yggdrasil/.claude/projects/-home-void-yggdrasil-CHAOS-DAO-FORGE/memory/celestial.db*
   ```
5. Restart the backend:
   ```bash
   cd /home/void-yggdrasil/CHAOS_DAO_FORGE/00_CORTEX/01_CREATION_REALM/Celestial_Blueprint/backend
   source .venv/bin/activate
   uvicorn app.main:app --reload --port 8787 &
   sleep 2
   ```
6. Confirm the DB is empty (0 blueprints):
   ```bash
   curl http://localhost:8787/api/blueprints | python -m json.tool
   ```
   Expected: `[]`
7. Rebuild:
   ```bash
   curl -X POST http://localhost:8787/api/rebuild | python -m json.tool
   ```
   Expected: `{"imported": N}` where N equals the number of `.md` files in `blueprints/`.
8. Confirm blueprints are back:
   ```bash
   curl http://localhost:8787/api/blueprints | python -m json.tool
   ```
   Expected: same list as before the DB was deleted.

## Acceptance Criteria
- [ ] After DB deletion and server restart, `GET /api/blueprints` returns `[]`
- [ ] `POST /api/rebuild` returns `{"imported": N}` with N ≥ 1
- [ ] `GET /api/blueprints` after rebuild returns all blueprints that had markdown files
- [ ] The canvas loads a recovered blueprint and shows its nodes
