# TASK T1.4.07 — `docs/API.md` endpoint reference

- **Depends on**: T1.4.02, T1.4.03, T1.4.04, T1.4.05, T1.4.06

## Context
A hard project rule: every endpoint is documented in `docs/API.md`. This is the contract the React client (T1.6.01), the future CLI (Stage 3), and the MCP server (Stage 4) all build against. The endpoints now exist across `blueprints.py`, `nodes.py`, `tracking.py`, `chronicles.py`, `io.py`. This task writes the human-readable reference for them.

## Objective
Write `docs/API.md` documenting every endpoint with method, path, body, and response.

## Instructions
1. Create `docs/API.md` with exactly:
   ```markdown
   ---
   title: "Celestial Blueprint — API Reference"
   tags: [api, reference, contract]
   status: active
   date_created: 2026-06-24
   ---

   # Celestial Blueprint API

   Base URL (dev): `http://localhost:8787`. All paths are prefixed `/api`.
   Responses are JSON unless noted. A "full blueprint" is the canonical
   blueprint object (see `docs/FORMAT.md` and `app/models/schemas.py::Blueprint`).

   Auth: in dev, none. Once T1.7.02 lands, every `/api/*` request (except
   `/api/health`) requires header `Authorization: Bearer <CELESTIAL_AUTH_TOKEN>`.

   ## Health

   | Method | Path | Body | Returns |
   |--------|------|------|---------|
   | GET | `/api/health` | — | `{"status":"ok","service":"celestial-blueprint"}` |

   ## Blueprints

   | Method | Path | Body | Returns |
   |--------|------|------|---------|
   | POST | `/api/blueprints` | `{name, tier?, realm?, summary?}` | full blueprint |
   | GET | `/api/blueprints` | — | `[{id,name,version,tier,status,updated_at}]` |
   | GET | `/api/blueprints/search?q=` | — | filtered summary list |
   | GET | `/api/blueprints/{id}` | — | full blueprint (404 if absent) |
   | PATCH | `/api/blueprints/{id}` | `{name?,status?,version?,summary?}` | full blueprint |
   | POST | `/api/blueprints/{id}/archive` | — | `{"status":"archived","id":...}` |

   `tier` ∈ {small, medium, large, continuous}. Create generates id
   `bp_{YYYY-MM-DD}_{slug}` and tier-default frontmatter.

   ## Nodes & Edges

   | Method | Path | Body | Returns |
   |--------|------|------|---------|
   | POST | `/api/blueprints/{id}/nodes` | `Node` | full blueprint |
   | POST | `/api/blueprints/{id}/edges` | `Edge` | full blueprint |

   `Node = {id, cat, type, title, body, data:{}}`. `type` is one of the 13
   node types in `docs/FORMAT.md`. `Edge = {src, dst, label}`.

   ## Tracking

   | Method | Path | Body | Returns |
   |--------|------|------|---------|
   | POST | `/api/blueprints/{id}/assumptions` | `Assumption` | full blueprint |
   | POST | `/api/blueprints/{id}/decisions` | `Decision` | full blueprint |
   | POST | `/api/blueprints/{id}/risks` | `Risk` | full blueprint |

   `Assumption = {id, bp_ref, status, risk, body}`,
   `Decision = {id, bp_ref, status, date, body}`,
   `Risk = {id, risk, likelihood, impact, mitigation, status}`.

   ## Chronicles

   | Method | Path | Body | Returns |
   |--------|------|------|---------|
   | POST | `/api/blueprints/{id}/chronicles` | `{kind, trigger?, cycle?, body}` | `{id, path}` |
   | GET | `/api/blueprints/{id}/chronicles` | — | `[chronicle rows]` |

   `kind` ∈ {raw, cycle}. The body is written to a markdown file under
   `chronicles/` and recorded in the DB.

   ## Import / Export / Recovery

   | Method | Path | Body | Returns |
   |--------|------|------|---------|
   | GET | `/api/blueprints/{id}/export` | — | markdown (text/plain) |
   | POST | `/api/blueprints/import` | `{markdown}` | full blueprint |
   | POST | `/api/rebuild` | — | `{"imported": N}` |

   `rebuild` re-imports every `.md` in `blueprints/` — the recovery path
   after a DB loss. Markdown is canonical; the DB is rebuildable.

   ## Error shape

   FastAPI default: `{"detail": "<message>"}` with the HTTP status code
   (400 parse error, 404 not found, 422 validation error).
   ```
