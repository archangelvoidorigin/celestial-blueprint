## Context
Assumptions, decisions, and risks are the "tracking" entities a planning session generates. This router exposes endpoints to add each, persisting via the store (T1.3.05) and rewriting markdown via `write_blueprint_file` (T1.4.02). Bodies validate against the `Assumption`, `Decision`, `Risk` models (T1.3.03). Like the other routers, each mutation returns the updated full blueprint.

## Objective
Create `api/tracking.py` with add-assumption/decision/risk endpoints and register it in `main.py`.

## Instructions
1. Create `backend/app/api/tracking.py` with exactly:
   ```python
   """Assumption, decision, and risk endpoints."""
   from __future__ import annotations

   from fastapi import APIRouter, HTTPException

   from app.models.schemas import Assumption, Decision, Risk
   from app.services import store
   from app.services.files import write_blueprint_file

   router = APIRouter(prefix="/api/blueprints", tags=["tracking"])


   def _require(bid: str) -> None:
       if not store.blueprint_exists(bid):
           raise HTTPException(status_code=404, detail="blueprint not found")


   @router.post("/{bid}/assumptions")
   def add_assumption(bid: str, item: Assumption):
       _require(bid)
       store.add_assumption(bid, item.model_dump())
       write_blueprint_file(bid)
       return store.get_blueprint(bid)


   @router.post("/{bid}/decisions")
   def add_decision(bid: str, item: Decision):
       _require(bid)
       store.add_decision(bid, item.model_dump())
       write_blueprint_file(bid)
       return store.get_blueprint(bid)


   @router.post("/{bid}/risks")
   def add_risk(bid: str, item: Risk):
       _require(bid)
       store.add_risk(bid, item.model_dump())
       write_blueprint_file(bid)
       return store.get_blueprint(bid)
   ```
2. Edit `backend/app/main.py`: under `# --- ROUTER INCLUDES ---`, ADD:
   ```python
   from app.api.tracking import router as tracking_router
   app.include_router(tracking_router)
   ```
