# TASK T1.7.03 — `dev.fish` final run script

- **Stage / Phase**: Stage 1 — Forge / Phase 1.7
- **Depends on**: T1.1.08
- **Produces**: `dev.fish` (minor update to copy `.env.example` if `.env` is absent)

## Context
`dev.fish` was created in T1.1.08 and already launches backend + frontend. Now that auth is in place (T1.7.01), the script should warn if `backend/.env` is absent and copy `.env.example` automatically — otherwise the developer gets a confusing 401 with no explanation. This is a small defensive update only.

## Objective
Update `dev.fish` to auto-copy `backend/.env.example` → `backend/.env` if `.env` is missing, and print a one-line notice.

## Instructions
1. Edit `dev.fish` at the **project root**: after the line `set FRONTEND $ROOT/frontend`, ADD these lines before the cleanup function:
   ```fish
   # Bootstrap .env if absent
   if not test -f $BACKEND/.env
       if test -f $BACKEND/.env.example
           cp $BACKEND/.env.example $BACKEND/.env
           echo "NOTE: copied .env.example -> backend/.env — edit CELESTIAL_AUTH_TOKEN before shipping."
       else
           echo "WARNING: backend/.env not found and no .env.example to copy."
       end
   end
   ```

## Acceptance Criteria
- [ ] `dev.fish` contains the `.env` check/copy block
- [ ] Running `fish dev.fish` (dry-run without starting servers) with no `.env` present produces the NOTE line
- [ ] An existing `.env` is NOT overwritten
