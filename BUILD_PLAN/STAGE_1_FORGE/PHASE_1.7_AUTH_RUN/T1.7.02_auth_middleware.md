# TASK T1.7.02 — Auth middleware + frontend token header

- **Stage / Phase**: Stage 1 — Forge / Phase 1.7
- **Depends on**: T1.7.01, T1.6.01
- **Produces**: edits to `backend/app/main.py` and `frontend/src/lib/api.ts`

## Context
`require_auth` (T1.7.01) is a FastAPI dependency that individual routes can depend on. The cleanest way to protect ALL `/api/*` routes at once is to add it as a router-level dependency at include time in `main.py`. The health endpoint stays open (it's at `/api/health` and included separately). The frontend sends the token from an environment variable injected at build time (`VITE_AUTH_TOKEN`).

## Objective
Apply `require_auth` globally to all routers (except health) in `main.py`, and add the token header to the frontend API client.

## Instructions
1. Edit `backend/app/main.py`: for each `app.include_router(...)` call (the five added by T1.4.02–06), add `dependencies=[Depends(require_auth)]`. Also import at the top. The result should look like:
   ```python
   from fastapi import Depends
   from app.core.auth import require_auth

   # in the ROUTER INCLUDES block:
   app.include_router(blueprints_router, dependencies=[Depends(require_auth)])
   app.include_router(nodes_router,      dependencies=[Depends(require_auth)])
   app.include_router(tracking_router,   dependencies=[Depends(require_auth)])
   app.include_router(chronicles_router, dependencies=[Depends(require_auth)])
   app.include_router(io_router,         dependencies=[Depends(require_auth)])
   ```
   The `/api/health` route defined directly on `app` is unaffected (no router).

2. Edit `frontend/src/lib/api.ts`: update the `req` function to include the token header when set. Replace the `headers` line:
   ```ts
   headers: { "Content-Type": "application/json" },
   ```
   with:
   ```ts
   headers: {
     "Content-Type": "application/json",
     ...(import.meta.env.VITE_AUTH_TOKEN
       ? { Authorization: `Bearer ${import.meta.env.VITE_AUTH_TOKEN}` }
       : {}),
   },
   ```

3. Document the variable in `frontend/.env.example` (create the file):
   ```
   # Frontend — copy to .env.local and set the token to match CELESTIAL_AUTH_TOKEN
   VITE_AUTH_TOKEN=
   ```

## Acceptance Criteria
- [ ] All five routers include `dependencies=[Depends(require_auth)]`
- [ ] `/api/health` still responds 200 without a token
- [ ] With an empty `CELESTIAL_AUTH_TOKEN`, all routes pass
- [ ] `frontend/.env.example` exists documenting `VITE_AUTH_TOKEN`
