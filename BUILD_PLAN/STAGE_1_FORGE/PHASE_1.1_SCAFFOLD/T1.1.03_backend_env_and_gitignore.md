## Context
The backend reads configuration from environment variables (DB path, auth token, port). Real secrets live in `.env` which must NEVER be committed. We commit only `.env.example` with placeholder values. A root `.gitignore` keeps secrets, venvs, build output, and the DB out of version control.

## Objective
Create the example env file and the project `.gitignore`.
